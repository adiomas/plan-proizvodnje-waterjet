Pristup generiranju PDF-a

Dva pristupa — oba daju isti rezultat za korisnika, razlika je u arhitekturi

A

Client-side PDF (@react-pdf/renderer)

Kako radi: React komponente definiraju PDF layout → generira se u browseru → download

Prednosti

  • Nema server-side koda — sve u browseru
  • React komponente za PDF layout — poznata tehnologija
  • Pixel-perfect kontrola nad dizajnom
  • Već ima tipove za TypeScript
  • Profesionalan izgled bez kompromisa

Nedostaci

  • ~200KB bundle size (gzip)
  • Sporije generiranje za veliki broj naloga
  • Vlastiti layout engine (ne CSS)
B

Server-side PDF (Next.js API Route + jsPDF)

Kako radi: API route prima parametre → generira PDF na serveru → vraća file

Prednosti

  • Ne opterećuje browser
  • Može se cachirati
  • Manji client bundle

Nedostaci

  • Potrebna API ruta + server logika
  • jsPDF nema React integraciju — imperativan API
  • Teže održavanje PDF layouta
  • Deployment overhead
Preporuka: Pristup A (@react-pdf/renderer) — deklarativni React pristup je čišći, lakši za održavanje, i daje profesionalniji rezultat. Bundle size od ~200KB je prihvatljiv za ovu vrstu aplikacije.
